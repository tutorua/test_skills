# Single UI Elements Test Scenarios Index

This directory contains test scenario files for each sub-item under the "Single UI Elements" category from the QA Practice site.

## Scenario Files

1. **[button.scenario.md](button.scenario.md)** - Test scenarios for basic clickable buttons, including click actions, keyboard interaction, and disabled states.

2. **[checkbox.scenario.md](checkbox.scenario.md)** - Test scenarios for single checkbox controls, covering toggle behavior, form submission, and accessibility.

3. **[input.scenario.md](input.scenario.md)** - Test scenarios for text input fields, including validation, paste operations, and character restrictions.

4. **[select.scenario.md](select.scenario.md)** - Test scenarios for single-select dropdown controls, focusing on option selection and keyboard navigation.

5. **[textarea.scenario.md](textarea.scenario.md)** - Test scenarios for multi-line text areas, covering line breaks, maxlength enforcement, and keyboard shortcuts.

6. **[radio.scenario.md](radio.scenario.md)** - Test scenarios for radio button groups, emphasizing exclusive selection and keyboard navigation.

7. **[toggle.scenario.md](toggle.scenario.md)** - Test scenarios for on/off toggle switches, including state persistence and accessibility features.

8. **[datepicker.scenario.md](datepicker.scenario.md)** - Test scenarios for calendar date-picker controls, covering date selection, validation, and localization.

9. **[slider.scenario.md](slider.scenario.md)** - Test scenarios for slider (range input) controls, focusing on value adjustment and boundary enforcement.

10. **[fileupload.scenario.md](fileupload.scenario.md)** - Test scenarios for file upload controls, including file type validation, size limits, and upload progress.

## Usage

Each scenario file follows the structure:
- **Requirements**: Key functional requirements derived from the QA Practice site
- **Positive tests**: Test cases that verify expected behavior
- **Negative tests**: Test cases that verify error handling and edge cases
- **Suggested missing tests**: Additional test cases not covered in the requirements

These scenarios are designed to provide comprehensive test coverage for UI element testing.
